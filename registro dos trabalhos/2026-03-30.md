# 🏁 Registro de Trabalho Robusto - 30/03/2026

Este documento detalha a grande reestruturação modular do banco de dados e a implementação da infraestrutura de automação (Webhooks) na **Ticketera (a2tickets360)**.

---

## 🛠️ O Que Foi Feito Hoje (Detalhamento Técnico)

### 1. Reestruturação Modular do Banco de Dados (Appwrite)
- **Problema**: A coleção `user_profiles` estava atingindo o limite físico de colunas (MariaDB Row Size) devido ao excesso de campos de onboarding (CNPJ, Bio, Endereços, Redes Sociais).
- **Intervenção**: Migramos os dados "pesados" para uma nova coleção dedicada chamada `organizer_details`.
- **Técnica**: 
    - `user_profiles`: Mantém apenas dados de identidade e status (`role`, `status`, `profileComplete`).
    - `organizer_details`: Armazena todos os metadados do produtor, vinculados pelo `userId`.
- **Resultado**: Sistema agora é escalável para centenas de novos campos sem risco de travar por limites de banco de dados.

### 2. Infraestrutura de Webhooks e Automação (N8N Ready)
- **Criação do Serviço**: Implementado o `webhookService.ts` para gerenciar disparos HTTP assíncronos.
- **Coleção de Configuração**: Criada a coleção `system_settings` para armazenar as URLs de destino (endpoints do N8N ou Zapier).
- **Gatilhos Implementados**:
    - `producer_registered`: Dispara automaticamente ao final do onboarding.
    - `producer_approved`: Dispara quando o Master Admin aprova o produtor.
    - `event_created`: Dispara quando um evento é publicado.
- **Segurança (ACLs)**: Ajustamos as permissões da coleção `system_settings` para permitir que o Administrador Master tenha controle total (CRUD) sobre as rotas de integração.

### 3. Interface Administrativa Master (UX/UI)
- **Webhooks Panel**: Criada a página `/master/webhooks` com interface premium.
- **Contraste e Legibilidade**: Refatoramos o estilo da tela para **Modo Claro (Light Mode)**, corrigindo um problema de legibilidade onde o texto branco ficava invisível sobre o fundo bege do painel Master.
- **Sidebars e Cards**: Injetamos a ferramenta de integrações tanto no menu lateral quanto na página principal do Administrador Master para facilitar o acesso.

### 4. Limpeza e Reset de Dados de Teste
- **Ação**: Deletamos todos os produtores "fantasmas" (com campos nulos) que travavam o fluxo de login/onboarding.
- **Verificação**: Confirmada a limpeza em `Auth`, `user_profiles` e `organizer_details`, permitindo um novo ciclo de testes limpo.

---

## 🧐 Desafios e Decisões
- **Permissões**: Enfrentamos um desafio onde os webhooks salvos não apareciam no front-end. O motivo era o bloqueio padrão do Appwrite (falta de permissão de leitura). Resolvemos forçando a atualização da coleção via script de provisionamento.
- **Truncamento**: Ajustamos os tamanhos dos campos `key` e `value` nas configurações do sistema para suportar URLs longas do N8N e chaves de evento descritivas.
